# Session 2 challenge box

Use this for quick oral checks or mini whiteboard tasks in class.

## 1) DictReader basics

Question: Why is `csv.DictReader` often easier to read than `csv.reader`?

Answer: It uses column names (`row["title"]`) instead of numeric indexes (`row[1]`), so the code is clearer.

## 2) String-to-number conversion

Question: Why can `int(row["votes"])` fail in real CSV files?

Answer: CSV values are strings, and some rows may be empty or invalid (for example `""`), which raises `ValueError`.

## 3) Safe average pattern

Question: What are the two values you must track to compute an average safely?

Answer: `total` and `count` (only increment count for valid values).

## 4) Missing cell detection

Question: How do you detect a missing field in a row dictionary?

Answer: Check for empty string values, for example `if row[col_name].strip() == "":`.

## 5) Break in search

Question: Why use `break` when finding the first row that matches a condition?

Answer: It stops early and avoids unnecessary iterations.

## 6) Reader comparison

Question: Give one case where `csv.reader` is still acceptable.

Answer: Simple scripts where index positions are stable and code is very short.

## 7) Complexity intuition

Question: Scanning all rows once is what time complexity?

Answer: `O(n)`.

## 8) Pairwise comparisons

Question: If you compare every response with every other response, what complexity do you get?

Answer: `O(n^2)`.

## 9) Binary search condition

Question: What must be true before applying binary search?

Answer: Data must be sorted.

## 10) Quick challenge

Question: Explain in one sentence why `O(n log n)` is usually better than `O(n^2)`.

Answer: Because `n log n` grows much more slowly than `n^2` as input size increases.
